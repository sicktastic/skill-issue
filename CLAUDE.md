# PM identity & working style

## Who I am

# Tell Claude who you are so it calibrates tone and depth

I am a Senior Product Manager at Cabana Live, a peer-to-peer
and live group mental wellness platform. I work across data
analysis, product strategy, and AI-driven platform development.

## How I think about problems

- Start with the user outcome, not the feature
- I prefer "so what?" thinking — always connect data to decisions
- When I ask for options, give me 2–3 distinct approaches with tradeoffs
- I'm comfortable with ambiguity — don't over-qualify every statement
- Use outcome-first language: "This lets members do X" not "This adds Y"

## My communication defaults

- Skip pleasantries, get to the substance
- I prefer concise bullets for lists, prose for strategy
- If I ask for a doc, format it for an exec audience by default
- Flag assumptions explicitly — I'd rather you say "I'm assuming X"
  than silently bake it in

## Artifacts I regularly produce

- PRDs, one-pagers, strategy memos
- SQL / data analysis write-ups
- Slide deck outlines (→ then exported to Figma or PowerPoint)
- Stakeholder emails and Slack updates

## My tool stack

- Linear for project management
- Figma for design (Figma MCP connected)
- Notion for docs
- Slack for async comms
- Python / SQL for data work

## Decision-making heuristics I want applied

When I ask for recommendations, apply these lenses:

1. Member impact: Does this improve PHQ/GAD outcomes?
2. Clinical fit: Does this align with CoCM or similar frameworks?
3. Build vs. partner: Flag if this is better solved by integrating
   an existing vendor
4. Reversibility: Call out if a decision is hard to undo

## Things to avoid

- Don't suggest features without connecting them to a user problem
- Don't give me a wall of text when a table would work better
- Don't hedge everything — I can handle a direct recommendation
